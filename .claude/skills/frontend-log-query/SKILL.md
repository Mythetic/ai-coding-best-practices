# 前端日志查询

通过 MCP 运维平台查询前端日志，将结果解析为格式化的逐行日志文件。

## 前置条件

- 环境变量 `MCP_AUTH_TOKEN` 必须已设置（从运维平台获取个人访问令牌）

## 执行流程

1. 从用户消息中提取 `aid`，支持以下三种格式：
   - **UUID 格式**：`58316f1b-023e-44f2-a297-2287db9ae71f`（8-4-4-4-12 位十六进制，用连字符分隔）
   - **openid格式**：`obLsS5TtZferfvIZ8PXUS0MJ0oqs`（固定28位，由大小写字母、数字、下划线、连字符组成）
   - **8位base36格式**：`a1b2c3d4`（固定8位，由小写字母a-z和数字0-9组成）
2. 从用户消息中提取时间范围：
   - **start_time**：开始时间，格式 `YYYY-MM-DD HH:MM:SS`
   - **end_time**：结束时间，格式 `YYYY-MM-DD HH:MM:SS`
   - 如果用户未指定具体时间，可根据用户描述推断（如"最近1小时"、"今天"、"昨天"等），使用当前时间计算
3. 确定查询参数：
   - **环境**：用户未指定时默认 `production`
4. 执行查询脚本（**直接传入本地时间**，脚本内部会自动转换为 UTC）：
   ```bash
   python scripts/query_frontend_log.py {aid} {start_time} {end_time} {环境} {当前工作目录}
   ```
5. 检查脚本输出：
   - 成功：告知用户日志文件保存路径
   - 失败：将错误信息反馈给用户

## 参数选择指引

### 环境选择

| 用户表述 | 参数值 |
|---------|--------|
| 线上、生产、正式环境（或未指定） | `production` |
| 联调、测试、开发环境 | `staging` |

### 时间范围推断

| 用户表述 | 推断方式 |
|---------|---------|
| 最近1小时 | end_time = 当前时间，start_time = 当前时间 - 1小时 |
| 最近30分钟 | end_time = 当前时间，start_time = 当前时间 - 30分钟 |
| 今天 | start_time = 今天 00:00:00，end_time = 当前时间 |
| 昨天 | start_time = 昨天 00:00:00，end_time = 昨天 23:59:59 |
| 指定了具体时间段 | 直接使用用户提供的时间 |

## 规则

### 必须

- 始终将当前用户的工作目录（workspace）作为第 5 个参数传入，确保日志保存在用户项目目录的 `logs/` 下
- 脚本输出失败信息时，完整转述给用户
- 时间范围不能超过 7 天，如果用户请求超过 7 天的范围，提示用户缩小范围
- 时间格式必须为 `YYYY-MM-DD HH:MM:SS`

### 禁止

- 禁止在未提取到有效 aid 的情况下执行脚本
- 禁止在未确定时间范围的情况下执行脚本
- 禁止猜测或编造 aid
- 禁止修改 `scripts/query_frontend_log.py` 脚本内容

## 输出

日志文件保存路径：`{工作目录}/logs/{aid}_frontend_formatted.log`
