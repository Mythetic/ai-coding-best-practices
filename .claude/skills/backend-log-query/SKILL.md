# 后端日志查询

通过 MCP 运维平台查询后端日志，将结果解析为格式化的逐行日志文件。

## 前置条件

- 环境变量 `MCP_AUTH_TOKEN` 必须已设置（从运维平台获取个人访问令牌）

## 执行流程

1. 从用户消息中提取 `requestId`，支持以下三种格式：
   - **UUID 格式**：`58316f1b-023e-44f2-a297-2287db9ae71f`（8-4-4-4-12 位十六进制，用连字符分隔）
   - **12位十六进制格式**：`ae1cebc6f0cf`（纯小写十六进制，精确 12 位）
   - **s前缀+数字格式**：`s1775184367984002616`（s 开头，后跟 19 位数字）
2. 确定查询参数：
   - **环境**：用户未指定时默认 `production`
   - **topic**：用户未指定时默认 `all`
3. 执行查询脚本：
   ```bash
   python scripts/query_backend_log.py {requestId} {环境} {topic} {当前工作目录}
   ```
4. 检查脚本输出：
   - 成功：告知用户日志文件保存路径
   - 失败：将错误信息反馈给用户

## 参数选择指引

### 环境选择

| 用户表述 | 参数值 |
|---------|--------|
| 线上、生产、正式环境（或未指定） | `production` |
| 联调、测试、开发环境 | `staging` |

### Topic 选择

| 用户表述 | 参数值 | 说明 |
|---------|--------|------|
| 所有日志、全部日志（或未指定） | `all` | 查询所有 topic，默认选项 |
| 应用日志 | `application` | 仅查询应用相关日志 |
| 网关日志、网关请求 | `gateway` | 查询网关层日志 |
| PDF 日志、PDF 处理、文档处理 | `pdf` | 查询 PDF 处理相关日志 |

## 规则

### 必须

- 始终将当前用户的工作目录（workspace）作为第 4 个参数传入，确保日志保存在用户项目目录的 `logs/` 下
- 脚本输出失败信息时，完整转述给用户

### 禁止

- 禁止在未提取到有效 requestId（不符合上述三种格式之一）的情况下执行脚本
- 禁止猜测或编造 requestId
- 禁止修改 `scripts/query_backend_log.py` 脚本内容

## 输出

日志文件保存路径：`{工作目录}/logs/{requestId}_formatted.log`
